Optimal sizing of infrastructure is a key requirement while planning a new data center or a cloud environment or a Kubernetes cluster, refurbishing an existing data center or optimizing your cloud or Kubernetes environment in response to ever increasing costs.

MapleLabs has built sizing solutions for ISVs/OEMs that are used by thousands of users worldwide to right size their infrastructure and save millions of dollars in costs.

What makes it challenging ?

A sizing exercise involves answering two questions:

  • Describing Workloads: in terms of CPU, Memory and Storage across disparate applications
  • Sizing: The most optimal configuration of CPU, Memory and Storage to run the workloads

There are thousands of combinations of hardware configurations that can be used to build a data center. Choosing the right combination becomes essential for predictable performance and optimize TCO.

The solution approach

There are usually two approaches to size workloads:

1. Profiling

Live apps are monitored for a few days to extract detailed performance trends. The data is then correlated and aggregate to extract metrics which is further utilized to right size the hardware components – be it CPU, Memory or Storage.

2. Modeling

Different types of applications such as Web Tier, Databases, VDI, Dev-Test or Clusters such as Splunk, Elasticsearch, Kafka or Spark etc. – all have different intrinsic characteristics which can be modeled through systematic performance tests. Using these models, it is possible to extrapolate the workload requirements based on usage parameters such as ingest rate , #queries per minute etc.

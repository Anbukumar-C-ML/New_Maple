Case Study

Building a Kubernetes based hyper scale application for field device management

Metrics from thousands of devices

The client has an install base tens of thousands of storage devices which routinely send diagnostic data. The client needed a scalable application that could connect to these devices and collect all diagnostic data. This data was then to be analyzed to provide actionable insights and speedy triaging through analytics and data visualization.

What was the challenge?

  • Streamlining massive bursts of data
  • Application scalability to accommodate 10s of Thousands of devices
  • Efficient Data Management – compression, storage and access at lower costsAnalytics & Reporting

MapleLabs solution

  • MapleLabs developed a container-based solution that was scalable with a focus on application security, reliability and cost control. The Kubernetes clusters run on AWS and utilizes SnappyFlow for monitoring and managing application performance and troubleshooting.
  • To improve overall application performance, individual pods were right sized with our sizing solutions. This enabled better on-demand scalability. Data bottlenecks were eliminated by streamlining data uploads that were occurring from multiple devices simultaneously reducing overall storge requirements.
  • The overall architecture was secure by design that included encryption, right tagging, RBAC and Kerberization features for authorized data access.

Solution architecture

  • Daily ingest : 2 TB of raw data parsed to 75 GB structured data and stored in Elasticsearch per day
  • Archive data : 2 TB per day
  • Ingest rate : 3000 requests at peak (transferring data of 250 MB to 3 GB per request)
